# 前端管理界面迁移说明

## 变更内容

### 已删除
- ✅ 后端静态HTML页面 (`internal/web/static/index.html`)
- ✅ 后端静态文件嵌入代码 (`//go:embed static`)
- ✅ 后端静态文件服务路由

### 已修改
- ✅ 后端根路径 (`/`) 现在返回API信息JSON，而不是HTML页面
- ✅ 后端日志信息更新，提示使用独立的前端服务
- ✅ README文档更新，说明前端是独立服务

### 新的架构

```
┌─────────────────┐         ┌─────────────────┐
│   前端服务      │────────▶│   后端API服务    │
│  (Next.js)      │  HTTP   │  (Go Server)     │
│  :8081          │  API    │  :8080           │
└─────────────────┘         └─────────────────┘
```

### 访问方式

- **前端管理界面**: http://localhost:8081
- **后端API服务**: http://localhost:8080/api
- **API根路径**: http://localhost:8080/ (返回API信息)

### 启动方式

使用提供的启动脚本：

```bash
./start-dev.sh
```

这将同时启动：
1. 后端API服务（端口8080）
2. 前端管理界面（端口8081）

### 优势

1. **前后端分离**: 前端和后端完全独立，便于开发和部署
2. **更好的开发体验**: 前端可以使用热重载、TypeScript等现代工具
3. **灵活的部署**: 前端可以部署到CDN，后端可以独立扩展
4. **现代化UI**: 使用Next.js和shadcn/ui构建的现代化界面

### 注意事项

- 后端不再提供HTML页面，只提供API服务
- 访问后端根路径 (`http://localhost:8080/`) 会返回API信息JSON
- 所有管理功能都需要通过前端界面访问
- 前端通过Next.js的rewrite功能代理API请求到后端
